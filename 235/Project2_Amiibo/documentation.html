Amiibo Finder Documentation

Overview

The Amiibo Finder is a web application that allows users to search for Amiibo figures, filter by game series, sort by name, and maintain a list of favourites. The application stores the last search term, selected filter, and sort option using the browser's local storage, providing a persistent and user-friendly experience.

Initial Build vs Final Rework

The initial build was a simple search interface similar to the GIF Finder project. The major reworks include:

  • Filter and sort functionality: Added dropdowns for game series and sorting by name (A→Z, Z→A).
  • Favourites feature: Users can add or remove Amiibo to a favourites section. IDs are stored in localStorage for persistence.
  • Persistent last search: The search term, filter, and sort selection are saved between visits.
  • Empty search handling: Unlike the initial build, users can perform searches using only filters without entering a search term.
  • Status messages: Added live feedback for search progress, number of results found, and error handling.

These reworks were necessary to align with assignment requirements, including minimum three controls, persistent state, and a responsive, user-friendly interface.

Challenges Encountered

  • Understanding the structure of the Amiibo API and how to use head and tail fields to create unique IDs for favourites.
  • Implementing search, filter, and sort in a way that could handle empty searches and persist user selections without using advanced JS constructs.
  • Ensuring the UI was clean and responsive using Flexbox and CSS Grid while keeping it simple and assignment-friendly.
  • Managing localStorage for multiple pieces of data (favourites, search term, filter, sort) and ensuring proper updates when the user interacts with the app.

Project Process & References

The project was developed in several stages to meet the assignment requirements:

  1. Initial Setup: Created a basic HTML layout with search input, results grid, and favourites section. This mimicked the starter template provided in class.
  2. API Integration: Explored the Amiibo API documentation to understand endpoints and data structure, including fields like name, gameSeries, image, head, and tail.
  3. Search Functionality: Implemented name-based search using the API. Initially, searches required a term, but this was later improved to allow filter-only searches.
  4. Filter and Sort: Added dropdowns for game series filtering and sorting by name (A→Z, Z→A), with persistent state using localStorage.
  5. Favourites: Enabled users to add or remove Amiibo from favourites, saved in localStorage for persistence across visits.
  6. Refinement: Added status messages, empty search handling, and responsive CSS layout with Flexbox and Grid to improve UX and match rubric requirements.

References & Resources Used:

  • Amiibo API Documentation - Primary reference for API endpoints, query parameters, and data structure.
  • Class Videos & Web Apps Tutorials - Guidance on search handling, localStorage, DOM manipulation, and assignment-specific requirements.

Conclusion

The final Amiibo Finder project demonstrates an improved, interactive user interface with persistent state, favourites management, and flexible search capabilities. The project was simplified enough to align with class expectations while still implementing key features using the API efficiently. This project highlights the importance of API documentation and careful planning in building feature-rich web applications.
